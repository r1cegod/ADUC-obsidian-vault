> **TL;DR**: Greg and Vin show the richest human workflow in the batch: load context with commands, use daily notes as raw reflection fuel, let commands surface patterns and ideas, and keep a strict wall between human-authored notes and agent-generated sidecar output.

## Summary
This source matters because it answers the human side of the vault problem. Instead of only showing ingest mechanics, it demonstrates context-loading commands, daily review commands, graph-aware traversal, idea generation, and the promotion of daily-note fragments into more durable artifacts. Its most important rule is the write boundary: the vault should stay rooted in what the human actually wrote, while agent-generated output should stay in a sidecar or compiled layer. This makes it the best source in the batch for note ownership, reflection, and command design.

## Key Points
- Commands like context, today, trace, connect, ideas, and graduate turn the vault into an active thinking partner.
- Daily notes are the raw stream that powers later analysis and idea generation.
- Backlinks and graph shape help the agent discover cross-domain connections.
- Reflection quality determines memory quality; the vault compounds only if the writing compounds.
- Human-written and agent-written material should not be mixed blindly.

## Details
### What the source teaches well
The strongest contribution is trust architecture. If the agent writes directly into the same notes it later analyzes, the vault can become a mirror of agent prose rather than the owner's actual thinking. This source provides the cleanest argument for sidecar synthesis and later graduation.

### Limits
The source is less formal than the Karpathy/Cole material on schema, lint, and ingest contracts. It is better for workflow and command ideas than for an exact maintenance spec.
