> **TL;DR**: This gist is a practical rules-drafting scaffold, useful for translating moderation intent into member-facing language, but it remains an unofficial template that needs local adaptation.

## Summary
The `infinotiver` rules template is a community-authored gist designed to make rule writing less painful for new server owners. It provides starter sections for profile-related behavior, general rules, and text-channel conduct, and explicitly advises keeping rules readable rather than excessively long. In this domain, its value is not authority but utility: it helps turn moderation systems into onboarding copy and public expectations. That makes it a bridge between internal policy and rule-screening text.

## Key Points
- Rule lists should be short enough that members actually read them
- Public rules need to cover both visible content and profile-level presentation issues
- Templates are useful starting points, but overcopying them can import rules that do not match the community

## Details
This source is best treated as raw drafting material. It can inspire rule categories and phrasing, but it should not be adopted wholesale because every community has different risk profiles, audiences, and tolerance thresholds. Its strongest contribution is simply forcing the question: what should be public, enforceable, and readable at the onboarding layer?
