> **TL;DR**: Use Canvas first for system-shape decisions, then mirror stable points into markdown so the vault remembers what the board discovered.

## When To Use
Use when the task changes system shape rather than function-local code logic.

## Routing In
- [[wiki/operations-hub]]
- [[wiki/synthesis/obsidian-d2-canvas-architecture-method]]
- project architecture hubs

## Steps
1. Route through the project README and architecture hub.
2. Update the active `.canvas` board.
3. Mirror stable decisions into markdown the same session.
4. Patch routers if the active board or architecture route changed.
5. Log the work before ending the response.

## Routing Out
- Drafting loop -> [[wiki/operations/draft-operation]]
- Maintenance closeout -> [[wiki/operations/self-healing-operation]]

## After Use Evolution Check
- Was Canvas really the right surface?
- Did the board stay human-readable?
- Did stable points get mirrored immediately?
- If friction appeared, patch this leaf or log the gap today.
