> **TL;DR**: The human-agent write boundary keeps trust in the vault by separating human-authored raw notes from agent-generated synthesis, so the system does not start reflecting its own writing back as if it were the owner's mind.

## Definition
This boundary says that human-primary notes should preserve the author's voice and raw thinking, while the agent should write mainly into compiled layers such as summaries, concept pages, synthesis pages, indexes, and logs.

## Key Ideas
- Raw notes and daily reflections should not be silently rewritten into agent voice.
- Durable takeaways should be promoted into sidecar or compiled pages instead of folded back into the raw note.
- Without a write boundary, the agent can start learning from its own outputs instead of the human's real thinking.
- Exceptions are allowed when the task explicitly asks for an update or when context files must be refreshed for operational accuracy.

## Applications
Use this when building personal second brains, executive-assistant systems, or any vault where authenticity and source ownership matter as much as retrieval quality.
