## North Star

```text
to build
  -> to become
    -> to be free
```

The operating goal is escape velocity: enough capability, proof, distribution, and optionality that Duc is not trapped by geography, credentials, weak networks, or cold lead dependence.

## Current Quarter Direction

Engineering:
- Keep PathFinder reviewable unless a real blocker threatens demo or review confidence.
- Build Raven as the next reusable AI-agent infrastructure project.

Knowledge system:
- Turn the vault into a reliable first-read brain for agents.

## 2026 Direction

- Build 2-3 shipped AI-agent projects that demonstrate full-stack depth: LangGraph, FastAPI, eval pipelines, memory, and usable workflows.
- Build inbound distribution around agent failure analysis, eval/replay demos, reusable scaffolds, and source-intelligence insights.
- Reduce cold-start cost for agents to near-zero: any session should reach productive state fast through Duc OS and project routers.
- Standardize a vault ingest pattern that scales across projects without turning memory into a landfill.

## Longer-Term Direction

- Reach Singapore / founder path with enough engineering depth and public proof to earn leverage.
- Ship AI systems that help real users, starting with Vietnamese students and then broader agent-tooling/product problems.
- Become strong at agent evaluation methodology, not just agent building.
- Operate as a Doer: ship -> measure -> refine, instead of extending planning cycles indefinitely.

## Distribution Thesis

Visibility is not vanity. It is leverage infrastructure.

```text
private building
  -> public artifact
  -> trust
  -> inbound attention
  -> better questions / collaborators / leads
  -> better products
```

The goal is not to never sell. The goal is to change the lead game from cold chase to trust capture, positioning, and conversion.

## Environment Strategy

The social problem is partly a room-density problem.

```text
low-density room
  -> shallow future/job talk
  -> Duc reaches for operating depth
  -> mismatch / collapse

higher-density room
  -> builders already shipping
  -> depth becomes useful sooner
  -> peers / collaborators / better questions
```

Do not over-optimize for belonging in rooms that cannot price the ambition correctly. Build proof, then use proof to enter denser rooms.

## World Velocity Rule

The world is too large and fast to track directly. Duc should not try to follow everything.

```text
world noise
  -> one chosen system
  -> filters / rankings / evals
  -> concentrated signal
  -> decisions
```

Raven exists because the answer is not more tabs. The answer is a personal signal engine.

## Direction, Not Ending

Escape velocity is not a final resting place. It is the point where the next move is chosen from leverage instead of constraint.

## Potential Scaffold Directions

### Raven / Knowledge Signal Engine

Raven filters public sources, especially YouTube and later Reddit/web sources, into signal. It formalizes Duc's taste and bullshit detector, extracts claims, scores novelty/actionability/evidence, writes durable summaries, and can later produce public synthesis.

The leverage is taste + evaluation + memory, not passive content hoarding.

### Agent Flight Recorder

A local-first trace/replay/eval scaffold for AI-agent builders. Useful later as infrastructure and portfolio proof, but not the immediate focus if the next goal is finding paid pain or sharpening Raven.

### Paid Pain Discovery Scaffold

A repeatable system to identify painful, already-monetized workflows, validate willingness to pay, and only then build the product or service.
