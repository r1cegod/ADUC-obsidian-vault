> **TL;DR**: This hub routes the maintenance and handoff workflows that keep PathFinder documentation, delegated work, and durable memory usable over time.

## Summary
Use this hub when the task is about how PathFinder work gets documented, handed off, or promoted into durable memory. It connects the delegated-feature protocol, the delegated human-facing docs themselves, the dev-log generation rules, and the context-maintenance discipline.

This cluster is smaller than evaluation, but formalizing it as a hub keeps workflow rules easy to find and keeps the project README from turning into a flat link dump.

## Suggested Entry Points
- Need to decide where new operational context belongs: [[projects/pathfinder/notes/docs-context-maintenance|Context Maintenance]]
- Need the authoring rules for delegated docs: [[projects/pathfinder/notes/docs-delegated-feature-how-to|Delegated Feature How-To Protocol]]
- Need the rule for what stays canonical in vault vs what mirrors back to repo: [[projects/pathfinder/notes/docs-repo-vault-sync-policy|Repo Vault Sync Policy]]
- Need a concrete delegated-doc example: [[projects/pathfinder/notes/docs-eval-run-eval|eval/run_eval.py - How To Use]], [[projects/pathfinder/notes/docs-evaluation-graph|Evaluation Graph]]
- Need the durable decision logging rule: [[projects/pathfinder/notes/docs-dev-log-system-prompt|Dev Log Generator - System Prompt]]
