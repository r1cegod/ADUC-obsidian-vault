# How To Write A Dev Log

Write like a human who needs a fast memory aid later, not like a generated report.

Rules:

- one file per day in `dev-log/days/`
- one line per day in `DEV_LOG.md`
- if the same day changes again, update that same day file
- keep it short
- no quotes around normal words
- no excessive context
- lead with what changed, why it mattered, and what is next

Recommended shape:

```md
# 2026-04-08

Summary: Short one-line day summary

## Updates

- changed:
- why:
- next:
```
