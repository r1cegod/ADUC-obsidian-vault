> **TL;DR**: If the Daniel Priestley long-form ingest reveals a coherent distribution system, keep raw evidence in Raven source lanes, compile the mechanism into a Raven Wave Report, then promote only the adopted operating model into Duc OS as a distribution engine.

## Growth Contract
- Parent branch: [[projects/raven/notes/raven-context-hub]] and [[projects/raven/notes/raven-tier-2-source-packet-contract]]
- Node role: leaf
- First parent link: [[projects/raven/notes/raven-context-hub]]
- Growth trigger: split when the distribution system accumulates repeated sub-systems such as offer architecture, authority positioning, content engine, partnership loop, sales conversion, or metrics dashboard.
- Forbidden contents: full transcripts, long source dumps, generic marketing advice, current project status, and unverified execution claims.
- Source/evidence boundary: Daniel Priestley transcript/source cards stay under Raven source lanes; this note defines the promotion structure only. Adopted Duc operating rules belong under Duc OS after evidence review.

## Promotion Stack

```text
Daniel Priestley raw transcripts
  -> projects/raven/sources/transcripts/<run>/yt/

per-video source cards
  -> projects/raven/sources/summaries/<run>/yt/raw/

cross-video Wave Report
  -> projects/raven/sources/summaries/<run>/yt/wave-report_daniel-priestley_distribution-system.md

compiled system map
  -> projects/raven/notes/raven-distribution-system-<slug>.md

adopted operating layer
  -> duc-os/distribution-engine.md only after Duc chooses to operationalize it
```

## Evidence Layers

### Layer 0 - Inventory

Every channel upload is accounted for. Shorts are marked `excluded_by_policy` for the first long-form ingest.

### Layer 1 - Source Card

One long-form video becomes one source card with:

```text
mechanism
claims
examples
failure modes
build implications
public artifact potential
```

### Layer 2 - Wave Report

The Wave Report answers:

```text
What is the distribution machine?
Which parts repeat across sources?
Where does Daniel contradict himself or update his model?
What is tactical vs strategic?
What is useful for Duc within 7-14 days?
```

### Layer 3 - System Map

If the Wave Report reveals a real system, extract it into a compact machine map:

```text
positioning
  -> audience / problem / category / wedge

offer
  -> pain / promise / mechanism / proof / risk reversal

authority
  -> books / talks / case studies / frameworks / visible proof

content
  -> idea capture / format / cadence / repurposing / distribution channel

conversion
  -> CTA / lead magnet / consult / product / community / partner route

feedback
  -> comments / DMs / calls / objections / sales data / content metrics

iteration
  -> what changes weekly
```

### Layer 4 - Duc OS Adoption

Only promote the parts Duc will actually run.

```text
source-derived model
  -> evidence-backed but external

Duc OS distribution engine
  -> adopted operating law for Duc's escape-velocity loop
```

## Anti-Traps

```text
transcript hoarding
  -> many saved words, no extracted machine

marketing mush
  -> generic advice without mechanism

guru capture
  -> copying Daniel's personality instead of stealing the system

premature adoption
  -> writing Duc OS rules before the Wave Report proves the pattern
```
