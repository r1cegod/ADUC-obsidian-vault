## Core Profile

```text
knowledge level: beginner / knows little about Wynncraft
primary mode: solo
optimization drive: high
class decision: Mage chosen for optimization despite melee preference
natural taste: melee / direct combat
accepted tradeoff: choose stronger practical solo route over preferred fantasy
assistant role: lead the conversation, pull sources, compile decisions, prevent drift
time-risk: avoid turning research/min-maxing into the real game
```

## Decision Bias

When tradeoffs appear, optimize in this order:

```text
1. solo survivability
2. scalable endgame relevance
3. clear progression path
4. efficient clear speed
5. low regret / low wasted investment
6. fun feel
7. aesthetic preference
```

Melee preference matters, but it does not override solo optimization unless Mage feels bad after real play.

## Strongest Definition

For Duc, strongest does not mean one stat.

```text
strongest practical build
  = survives solo
  + kills fast enough
  + has mobility/control
  + does not require impossible gear too early
  + remains relevant after patch drift
  + avoids expensive mistakes
```

## Assistant Behavior

Default behavior:
- lead with one recommendation
- ask for minimal state only when needed
- run [[projects/wynncraft/notes/wynncraft-search-operations]] for current/drift-prone questions
- use community-heavy search for builds, then official confirmation
- update [[projects/wynncraft/notes/wynncraft]] when a decision changes the route

Avoid:
- giant option dumps
- pure DPS fantasy without survival/cost checks
- stale build reuse
- telling Duc to research communities manually
- API/data-platform work unless explicitly promoted
